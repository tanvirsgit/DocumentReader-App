3. Design Approach


The document reader with an eye tracking system will be built using object oriented concepts and principles. Various classes and their objects will interact with each other to provide the proper functionalities. In object oriented design concept, there are two generic and most commonly used approaches: “top-down” approach and “bottom-up” approach. In this system we are going to implement the bottom up approach to construct the whole system.



Bottom Up approach is best suitable for developing systems that already have some existing components that can be used to build the new system. In this system, the eye tracking feature is going to be implemented using PyGaze (an open source library for detecting eye tracking) for the web application. For the mobile application(Android), Mobile Vision API (official android vision API). So to use these existing libraries as a component for our system we have chosen the bottom up approach

We are going to build all the small components and then they will form bigger components which will eventually produce the desired application. This approach will enable us to test each component individually for potential errors and correct them as we build the system. It will help to identify crucial decisions taken during development of the application as it is progressing.

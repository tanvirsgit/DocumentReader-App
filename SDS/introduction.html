    Software Design Specification

1. Introduction

1.1 Purpose of the document


The purpose of this document is to present the system design of the software Document Eye Tracker by which one can determine the extent of how much a document is read. The software will have two types of user: Faculty and Student. A database will store the document(s) uploaded by the faculty in which will be identifiable against the faculty's initials. The uploaded document is available for viewing once a student has logged in and accessed the documents page for a particular faculty within the software. Using an image recognition framework, once a document is read from within the software, it will track the user's pupils and estimate within a certain threshold for determining attention. Along with gaze data, the time spent looking at a document will help determine a better report of gaze and attention a student has spent in reading the document. A report is generated against each individual student accounts if an assigned document has been issued by a faculty, which the faculty can access from his/her panel.

1.2 Scope of project


The software is aimed at solving the issue of tracking a student's academic performance by analyzing their study patterns. This will help faculties and teaching fellows understand to a greater depth how a student manages assigned tasks such as homeworks and how much time they actually spend in reviewing study materials. These type of metrics will be benefial to researchers analyzing studying patterns and help genearate better statistics.

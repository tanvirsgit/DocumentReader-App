3. External Interface Requirements


3.1 User Interfaces

The system will be implemented using a website and an android application where the users will be able to read pdf. When user opens the software for the first time he/she will be asked to register to the system using a federated authentication service preferably from Facebook/Gmail or register a new account using email/password. So the first screen that will appear is the authentication page.

Or if the user opts to sign-up using a federated authentication service such as Facebook/Gmail.



If a login is not successful, an error message is presented to the user.


After successful login, user will be taken to a page where there will be some basic information about the user(account name) and options on the navbar. For the website, it will look like a standard HTML dashboard page. In the Android application the necessary elements will be in textViews, imageViews, etc. The menu items will be presented with a sidebar. Users will be able to download PDFs through a download button in both platforms. After downloading, an integrated PDF viewer(tracker enabled) will render the PDF from within the software.

The user will be asked to grant the device to access its camera.



3.2 Hardware Interfaces

In this project, the system uses cameras for eye tracking. So in both platforms the system will need to interact with the camera(front facing in mobile devices/webcam in PCs). For the website, the system would use a 3rd party API which enables access to the webcam. The browser and the underlying operating system will manage protocols and permissions to access the camera. For the android application, camera access will be handled by android standard API for access grant/usage.


3.3 Software Interfaces

The system uses a central server in conjunction with a database to handle functionality. The central server is responsible for processing and analyzing the image patterns and storing a detailed periodic report on the database. When a new user has registered, the database is updated with said credentials. Also when a PDF download is requested, the server sends it over to that requested platform. The website and Android app sends over data via the internet at periodic intervals so that the server can generate a report.


3.4 Communication Interfaces

The communication between the different parts of the system is important since they depend on each other. The website and android application both need to communicate with the server in order to download PDFs and upload user reports. For that HTTP protocol is used to communicate between the components over the internet. However, the communication medium is handled by the underlying operating systems and is of less concern to the software itself provided that it is working correctly to perform its functions.

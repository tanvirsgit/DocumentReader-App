1. Introduction


1.1 Purpose

The purpose of this document is to provide a brief overview for an unique type PDF reader which will monitor and record statistics of a document linked with an account. It will generate data, and will report to a governing body of how much the document was read and the necessary timelines along with a detailed report accompanied with graphs. The aim is to make a software that is simple in design, intuitive and does what it’s intended.


1.2 Document Conventions

The document text size will be visible enough to cater to a broad audience irrespective of age.The preferred choice of font is “Sans Serif” and a color scheme that is pleasing to the eyes.


1.3 Intended Audience and Reading Suggestions

This document is intended for developers, users, testers and project managers. The document emphasizes on the product specifications and as such is mainly intended for people wanting to know about the features and future plans about the product.


1.4 Product Scope

The main purpose of this software is to keep track of a user’s reading and or an assigned progression of a reading material. The software will tell the user when a certain document was last opened and how far the document was read over multiple platforms. It will record data(image analysis) frequently and have a server process that information, upon next viewing of the document the previous records will be displayed for the user’s convenience. Behind the scenes, it is going to log a detailed report(statistics) basing upon the metrics of the data stored and report to admins/teachers.
